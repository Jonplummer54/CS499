Professional Self-Assessment

Completing the Computer Science program at SNHU and developing this ePortfolio have given me the opportunity to consolidate the technical knowledge and practical skills I have built throughout the program and connect them to the professional experience I bring from my current role as a data visualization specialist. Working alongside data scientists daily has given me a firsthand perspective on how software engineering, data structures, algorithmic thinking, and database design come together to deliver analytical tools that inform real decisions. The coursework in this program provided the formal computer science foundation to complement that experience, and this capstone project gave me the space to demonstrate that growth through a single artifact enhanced across three categories of the discipline.

Collaborating in a Team Environment

Throughout the program, I developed a strong understanding of how collaborative software development works in practice. In CS 250 Software Development Lifecycle, I learned the Agile and Scrum frameworks that structure how development teams plan, execute, and deliver software iteratively. These concepts were not abstract to me because in my professional role, I work within cross-functional teams where data scientists, analysts, and engineers collaborate on shared deliverables. Understanding how to break work into sprints, participate in standups, and incorporate stakeholder feedback are skills I apply daily.

In this capstone, I demonstrated collaboration readiness through the code review process. Producing a code review video required me to explain my existing code to a non-technical audience, identify weaknesses honestly, and articulate a clear plan for improvement. I also wrote comprehensive Javadoc documentation throughout my artifact so that any developer joining the project could understand the codebase without requiring a walkthrough from me.

Communicating with Stakeholders

Effective communication is something I practice professionally every day. As a data visualization specialist, my primary responsibility is translating complex data into visual formats that stakeholders can understand and act on. This skill translated directly into the capstone in several ways. The narratives I wrote for each enhancement required me to explain technical decisions in clear, structured prose. The code review video required me to present technical material orally, adapting my explanation to an audience of peers rather than deeply technical specialists.

Within the artifact itself, the GoalTrackerActivity demonstrates communication through design. The linear regression algorithm produces a slope value measured in pounds per day, but displaying a raw slope to the user would be meaningless. Instead, the activity translates algorithmic output into user-friendly language: trend direction is displayed as "Losing weight," "Gaining weight," or "Stable," and the goal projection is displayed as a calendar date rather than a number of days.

Data Structures and Algorithms

CS 260 Data Structures and Algorithms was one of the most foundational courses in the program for me. It gave me the vocabulary and analytical framework to evaluate algorithmic choices systematically rather than relying on intuition. In this capstone, I applied those skills by implementing five distinct algorithms and data structures: linear regression for trend analysis, a simple moving average with a LinkedList sliding window for data smoothing, binary search for efficient date lookup, a HashMap index for constant-time existence checks, and a TreeMap for sorted date-weight mappings. Each choice involved explicit trade-offs that I documented in the code.

Software Engineering and Databases

CS 340 Advanced Programming Concepts and DAD 220 Introduction to SQL taught me how to design software systems that are maintainable, extensible, and data-driven. In this capstone, I applied software engineering principles by refactoring the original artifact from a tightly coupled prototype into a modular, maintainable application. On the database side, I enhanced the SQLite schema with foreign key constraints and cascading deletes, created composite indexes, implemented a non-destructive migration strategy, and added strict date validation at the database layer.

Security

Developing a security mindset was a recurring theme throughout the program. The capstone artifact demonstrates security practices at multiple layers. User passwords are hashed using SHA-256 with a unique cryptographically random salt per user. Input validation is applied defensively at both the UI layer and the data access layer, implementing defense in depth. The database enforces NOT NULL constraints and foreign key constraints to maintain data integrity. Weight values are range-checked and dates are validated using strict parsing to reject impossible calendar dates.

Portfolio Summary

The artifact in this ePortfolio is the Weight Tracking App, an Android mobile application originally developed in CS 360 Mobile Architecture and Programming. I chose to enhance a single artifact across all three categories — software design and engineering, algorithms and data structures, and databases — because doing so demonstrates the depth of improvement possible when computer science principles are applied systematically to an existing codebase.

Together, the three enhancements demonstrate that I can take a working piece of software and elevate it to professional standards by applying the principles learned throughout the Computer Science program. Explore each enhancement below.
