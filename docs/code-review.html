Existing Functionality

The original Weight Tracking App was developed in CS 360 Mobile Architecture and Programming. It was a native Android application built with Java and SQLite that provided the following basic features:

  • User Authentication: Registration and login with plaintext password storage
  • Weight Logging: Add and delete daily weight entries displayed in a RecyclerView
  • SMS Permissions: A dedicated activity for requesting SEND_SMS permission
  • Database: Two tables (users and weight_log) with no constraints or indexes

The app was functional but lacked analytics, search capability, input validation, security, testing, and documentation.

Code Analysis

The code review identified the following areas for improvement:

Structure & Design

The RecyclerView adapter was tightly coupled to the activity. The data model used mutable fields. Permission logic was duplicated across activities rather than centralized.

Functionality

No analytical features existed. Users could only log and view weights with no insight into trends, progress, or projections. There was no way to search for entries or export data.

Efficiency

No search functionality beyond scrolling. No database indexes, resulting in full table scans for every query.

Security

Passwords stored in plaintext. No input validation on weight values or dates. No database constraints beyond basic column types.

Testing & Documentation

No unit tests or instrumented tests existed. Minimal code comments with no Javadoc documentation.

Enhancement Plan

The code review outlined enhancements across three categories, mapped to the five course outcomes:

Enhancement 1: Software Design & Engineering

  • Refactor to immutable data model
  • Introduce callback interfaces for loose coupling
  • Centralize permission handling
  • Add CSV data export with dual storage path support
  • Implement comprehensive input validation
  • Add try-with-resources for resource management
  • Write Javadoc documentation throughout

Enhancement 2: Algorithms & Data Structures

  • Implement linear regression for trend analysis
  • Implement moving average with LinkedList sliding window
  • Add binary search for date lookups
  • Build HashMap date index for O(1) existence checks
  • Use TreeMap for sorted date-weight mappings
  • Add goal projection and progress calculation
  • Write comprehensive unit tests
